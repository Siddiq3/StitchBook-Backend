🎯 AUTHENTICATION FLOW
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1️⃣  User enters phone number
2️⃣  Firebase sends OTP SMS
3️⃣  User enters OTP code
4️⃣  Firebase returns ID Token
5️⃣  POST /api/auth/login { firebaseToken }
6️⃣  Backend verifies token with Firebase Admin SDK
7️⃣  Extract phone from token (NEVER trust frontend)
8️⃣  Normalize phone to E.164 format (+919876543210)
9️⃣  Find existing user by phone OR create new user
🔟 Issue JWT token (30 days expiry)
1️⃣1️⃣ Return: { token, refreshToken, user, expiresIn }
1️⃣2️⃣ Frontend stores JWT in AsyncStorage
1️⃣3️⃣ All requests include: Authorization: Bearer JWT_TOKEN

├─ Step 1: Use Firebase for OTP (not backend)
├─ Step 2: Get Firebase ID Token
├─ Step 3: POST /api/auth/login { firebaseToken }
├─ Step 4: Save JWT from response
└─ Step 5: Use JWT for all requests

BACKEND (Node.js):
├─ Firebase token verified by Firebase Admin SDK
├─ Phone extracted from token (trusted source)
├─ User found/created by phone (unique identifier)
├─ JWT issued (30 days)
└─ Protected endpoints check JWT with authMiddleware

DATABASE:
├─ Phone is UNIQUE
├─ auth_provider tracks OTP method
├─ shop_id links user to business
└─ Fully backward compatible
