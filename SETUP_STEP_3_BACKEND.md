## 🎮 STEP 2: Start Development Server

### Option A: Quick Start (Recommended)

```bash
npm run dev
```

**What this command does:**
- Starts the server with nodemon (auto-reload)
- Listens on http://localhost:5000
- Restarts automatically when files change
- Perfect for development

**Expected output:**
```
✓ Server started successfully
✓ Environment: development
✓ Server running on http://localhost:5000
✓ Health check: http://localhost:5000/health
```

### Option B: Production Start

```bash
npm start
```

**Differences:**
- Runs without auto-reload
- Doesn't watch for file changes
- Used for production deployment
- Manual restart needed after code changes

## 🆘 TROUBLESHOOTING

### Issue 1: "npm: command not found"
```
Solution: Node.js not installed
1. Download from https://nodejs.org/
2. Install (includes npm)
3. Restart terminal
4. Verify: node --version
```

### Issue 2: "Port 5000 already in use"
```
Solution: Another app using port 5000

Option A: Stop other app
lsof -i :5000              # Show process using port 5000
kill -9 <PID>              # Kill process (replace PID)

Option B: Use different port
Edit .env:
PORT=5001
npm run dev
```

### Issue 3: "DATABASE_URL is undefined"
```
Solution: .env file missing or incomplete
1. Check .env exists: ls -la .env
2. Check DATABASE_URL is in .env
3. Restart server: npm run dev
```

### Issue 4: "Error: connect ECONNREFUSED"
```
Solution: Database not running or connection wrong
1. Verify database is running:
   - Supabase: check online at supabase.com
   - Local: psql -U postgres
2. Verify DATABASE_URL in .env
3. Test connection: psql $DATABASE_URL
```

### Issue 5: "Cannot find module 'express'"
```
Solution: node_modules not installed
1. Run: npm install
2. Wait for completion
3. Run: npm run dev
```

### Issue 6: "Module not found: middleware/auth"
```
Solution: File path issue in require()
1. Check file exists: ls src/middleware/auth.js
2. Verify spelling matches exactly
3. Restart server
```

### Issue 7: "EACCES: permission denied"
```
Solution: Permission issue (rare, usually on Linux)
Option A: Fix permissions
sudo chown -R $(whoami) ~/Desktop/studygargae/tailor-backend

Option B: Reinstall globally
npm install -g npm
npm install
```

## 📊 Common Error Messages & Solutions

| Error | Cause | Solution |
|-------|-------|----------|
| PORT 5000 already in use | Another app using port | Change PORT in .env |
| ECONNREFUSED database | Database not running | Start PostgreSQL/Supabase |
| MODULE_NOT_FOUND | Missing dependency | Run `npm install` |
| JWT_SECRET is undefined | .env not loaded | Restart server |
| JSON.parse error | Config file corrupt | Check .env format |
| TypeError: Cannot read | Syntax error | Check syntax in .env |

## 🎯 What Each Command Does

### npm install
```
Reads: package.json
Downloads: All packages from npm registry
Installs: Into node_modules/
Creates: package-lock.json
Time: 2-3 minutes

Use: Only first time or after adding new packages
```

### npm run dev
```
Runs: src/server.js with nodemon
Watches: All .js files for changes
Auto-restarts: On file save
Port: 5000 (or value in .env)
Logs: All requests and errors

Use: Every time during development
```

### npm start
```
Runs: src/server.js directly
No watch: Doesn't auto-restart
No logs: Minimal logging
Port: 5000 (or value in .env)

Use: Production deployment only
```

## 📈 Performance Tips

### Optimize npm install
```bash
# Install with legacy peer deps (if errors)
npm install --legacy-peer-deps

# Install with no-optional (faster)
npm install --no-optional

# Clear npm cache (if corrupted)
npm cache clean --force
npm install
```

### Speed Up Server Startup
```bash
# Install globally for faster access
npm install -g nodemon

# Clear node_modules and reinstall
rm -rf node_modules package-lock.json
npm install
```
